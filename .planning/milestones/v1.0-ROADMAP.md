# Milestone v1.0: PDF Pipeline Demo

**Status:** SHIPPED 2026-02-18
**Phases:** 1-3
**Total Plans:** 5

## Overview

This project proves that customer-provided Word templates can be converted to HTML and rendered as pixel-accurate PDFs with dynamic data. The roadmap moved from establishing a working PDF generation pipeline (Maven, fonts, QR codes, Thymeleaf), through building two complete demo templates with print-ready CSS, to a verification phase with JUnit tests that produce and assert on generated PDFs.

## Milestone Summary

**Key Decisions:**
- OpenHTMLtoPDF 1.1.37 with `io.github.openhtmltopdf` group ID (migrated from `com.openhtmltopdf`)
- DejaVuSans 2.37 font embedded via temp file extraction (builder.useFont requires File)
- useFastMode() enabled on PdfRendererBuilder
- CSS 2.1 only: no Flexbox, no Grid, table-based layouts required
- Standalone ThymeleafRenderer wrapping SpringTemplateEngine (no Spring Boot auto-config)
- Raw Open XML ZIP for .docx artifacts (no Apache POI dependency)
- Table-based 2-column layout with border-left gutter (CSS 2.1 paged media compatible)

**Issues Resolved:**
- GitHub raw URLs for DejaVuSans.ttf returned HTML (Git LFS) — resolved via SourceForge download
- CSS link paths needed to be relative to base URI root, not template file location

**Issues Deferred:**
- @page margin boxes with running() elements not runtime-verified — may need CSS fallback

**Technical Debt Incurred:**
- None significant — clean demo codebase
